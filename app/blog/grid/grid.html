CSS Grid Support

One of the advantages of css Grid is the ease it provides web developers to lay out items on a website. As of today, the current browser support for CSS Grid is at 68.14%. However, this does not reflect the current browser market share, nor does it reflect which versions users currently have installed.

We are only interested in the browser versions that support CSS Grid.

  • Edge 16 - 0%
  • Firefox 54 - 2.94%
  • Chrome 59 - 5.74%
  • Safari 10.1 - 0.21%
  • Opera 47 - 0.36%
  • iOS Safari 10.3 - 8.41%
  • Android Browser 56 - 0%
  • Chrome for Android 59 - 29.53%

Each of those represent the earliest version that has support for CSS grid.

Looking at the current trends, as of Setpember 2017, 23.79% of the market is using Google Chrome 60.0. But in second place of the market share we have Safari Mobile 10 that does not support CSS Grid, at 16.31%. It shouldn't be a problem since we should be designing for mobile first. And mobiles have less real estate area.

Finally, we have Firefox 55 at 13.6%;

What is Grid

  • Fixed and flexible track sizes: % or fr
  • Item placement: You can place items using line number, names or by targeting an area of the grid.
  • Creation of additional tracks to hold content
  • Alignment control: We can control how items are aligned once placed on the grid area, and how the entire grid is aligned.
  • Control of overlapping content: More than one items can be placed into a grid cell, or areas can partially overla each other. This layering may then be controlled with z-index.
