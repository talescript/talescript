Why I chose eleventy

August 25, 2019

I won't rehash what has already been said between the differences of a static site and a dynamic one. There is already a lot of info on the subject.

But I will say that managing a static site is both cheaper and easier, but it is geared towards people with a modicum of technical experience. Unless there is something you would like to change, the default settings can get you pretty far - which in turn deminishes the amount of technical experience needed.

We have the usual recommendations:

  • Jekyll
  • Hugo
  • Pelican

Each has it's pro's and cons, and each written in a different programming language. Each with their learning curve. Just like Wordpress. If you've never used Wordpress, you end up in pretty much what is the wild west. It's a difficult place to navigate with so many people trying to grab your attention.

After having gone through the different stages, first looking through Wordpress, followed by the usual static recommendations, I ended up chosing eleventy.

At the moment Eleventy has the right trade-offs. It's small enough to get out of your way and comes with sensible defaults. But it does require some configuring. But this is what, from a personal level, enjoy about this static site generator. It's ideal for beginners to get their hands dirty and start figuring things out. With older static site generators that have been around for a while, you end up reading the documentation. And then comes the coin toss, whether that documentation is digestible or not.

All in all, Eleventy is as easy as:

npm init -y
npm install @11ty/eleventy --save-dev
echo '# Page header' > README.md
npx @11ty/eleventy

Eleventy will build the pages under _site/README/index.html

You can view them using in your localhost running the command npx @11ty/eleventy --serve

And as easy as that you can get quite creative on what you create.
